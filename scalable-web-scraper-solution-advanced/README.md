# Web Scraping Tools System

A comprehensive, production-ready full-stack web application for defining, running, and managing web scraping tasks. This system is built with FastAPI (Python) for the backend, React for the frontend, PostgreSQL for the database, and uses Playwright for robust web scraping. Celery with Redis handles asynchronous scraping jobs, ensuring scalability and responsiveness.

## Features

**Core Application:**
*   **Scraper Definition:** Define new scrapers with a name, starting URL, and JSON-based parsing rules (CSS selectors for items, fields, and optional pagination).
*   **Scraping Job Management:** Trigger scraping jobs manually, track their status (pending, running, completed, failed), and view results.
*   **Scraped Data Storage:** All extracted data is stored in the database, linked to its scraper and job run.
*   **API Endpoints:** Full CRUD operations for Users, Scrapers, Jobs, and Scraped Data.
*   **Frontend UI:** Intuitive React interface for managing all aspects of the system.

**Database Layer:**
*   PostgreSQL for relational data storage.
*   SQLAlchemy ORM for Python interactions.
*   Alembic for database migrations.
*   Seed data for an initial superuser.

**Configuration & Setup:**
*   `requirements.txt` (Python) and `package.json` (Node.js) for dependency management.
*   Environment variables for sensitive information and configuration.
*   Docker and Docker Compose for containerization and orchestration, simplifying local development and deployment.
*   CI/CD pipeline configuration using GitHub Actions (example provided).

**Testing & Quality:**
*   **Unit Tests:** `pytest` for backend logic, `jest`/`react-testing-library` for frontend components (coverage aimed at 80%+).
*   **Integration Tests:** API tests simulating client interaction.
*   **Performance Tests:** Example `locust` script for load testing the backend APIs.

**Additional Features:**
*   **Authentication & Authorization:** JWT-based authentication with role-based access control (superuser/normal user).
*   **Logging & Monitoring:** Structured logging for backend operations, Celery tasks, and error handling.
*   **Error Handling Middleware:** Centralized error handling for consistent API responses.
*   **Caching Layer:** `async_lru` for local API response caching, Redis integration for `FastAPILimiter` and Celery.
*   **Rate Limiting:** Protects API endpoints from abuse using `fastapi-limiter` with Redis.
*   **Playwright:** Handles modern JavaScript-heavy websites, browser automation, and headless execution for robust scraping.
*   **Celery:** Asynchronous task queue for running long-running scraping jobs in the background, preventing API timeouts.
*   **Flower:** Web-based tool for monitoring and administering Celery tasks.

## Setup and Installation

### Prerequisites

*   Docker and Docker Compose installed.
*   Git installed.
*   A text editor (VS Code recommended).

### 1. Clone the repository

```bash
git clone https://github.com/your-username/web-scraping-system.git
cd web-scraping-system
```

### 2. Configure Environment Variables

Create a `.env` file in the root directory of the project by copying from `.env.example`:

```bash
cp .env.example .env
```

Edit the `.env` file and replace `YOUR_SUPER_SECRET_KEY_FOR_JWT` with a strong, random string (e.g., generated by `openssl rand -hex 32`). You can keep other defaults for local development.

### 3. Build and Run with Docker Compose

From the root directory of the project, run:

```bash
docker-compose build
docker-compose up -d
```

This will:
*   Build Docker images for the backend, frontend, Celery worker, and Flower.
*   Start PostgreSQL, Redis, Backend API, Celery worker, Celery Flower, and Frontend services.
*   Apply database migrations and seed the initial superuser (defined in `.env`).

**Wait for Services to Start:** It might take a few minutes for all services to be healthy, especially the database and backend. You can check the status with `docker-compose ps`.

### 4. Access the Applications

Once all services are up:
*   **Frontend (Web UI):** `http://localhost:3000`
    *   Login with the superuser credentials defined in your `.env` (default: `admin@example.com` / `adminpassword`).
*   **Backend API (Swagger UI):** `http://localhost:8000/api/v1/docs`
*   **Celery Flower (Task Monitor):** `http://localhost:5555`

### 5. Stop the Services

```bash
docker-compose down
```
To remove volumes (and thus all database data):
```bash
docker-compose down -v
```

## Running Tests

### Backend Tests

1.  Ensure Docker Compose is running (`docker-compose up -d`).
2.  Connect to the backend container: `docker-compose exec backend bash`
3.  Inside the container, run pytest:
    ```bash
    pytest --cov=app --cov-report=term-missing tests/
    ```
    (Exit the container with `exit`)

### Frontend Tests

1.  Connect to the frontend container: `docker-compose exec frontend bash`
2.  Inside the container, run jest:
    ```bash
    npm test
    ```
    (Exit the container with `exit`)

### Performance Tests (Locust)

1.  Ensure all Docker Compose services are running (`docker-compose up -d`).
2.  From your host machine, navigate to the `backend` directory: `cd backend`
3.  Install Locust: `pip install locust`
4.  Run Locust: `locust -f tests/performance/test_performance.py`
5.  Open your browser to `http://localhost:8089` and start the test.

## Development

### Backend Development

If you wish to develop the backend outside Docker (e.g., for faster iteration with IDE debugging):

1.  Ensure `db` and `redis` services are running via Docker Compose: `docker-compose up -d db redis`
2.  Create a Python virtual environment: `python -m venv .venv`
3.  Activate it: `source .venv/bin/activate` (Linux/macOS) or `.\.venv\Scripts\activate` (Windows)
4.  Install dependencies: `pip install -r requirements.txt`
5.  Apply migrations: `alembic upgrade head`
6.  Run the FastAPI app: `uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload`
7.  Run Celery worker (in a separate terminal): `celery -A app.tasks.scraping_tasks worker --loglevel=info`

### Frontend Development

1.  Ensure the backend service is running (either via Docker or locally).
2.  Navigate to the `frontend` directory: `cd frontend`
3.  Install Node.js dependencies: `npm install`
4.  Start the React development server: `npm start`
    The app will typically open in your browser at `http://localhost:3000`.
